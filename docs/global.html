HostContext

Includes options passed to the model compiler as well as data passed along to the platform-specific resource-fetching API.
Type:
  • object
Properties:
Name Type Attributes Default Description
reportAllErrors boolean <optional>
 false Indicates whether or not data model (aka JavaScript) compilation errors cause the callback to be invoked with errors. By default these errors are raised to the offending document as error.execution events.
deferCompilation boolean <optional>
 false Generate a single model string with the datamodel compiled in as local variables. This can be helpful for inspecting the generated code, as well as dumping a file to the filesystem that can be subsequently required. In particular, this is used to work around this nodejs bug related to debugging generated code.
strict boolean <optional>
 false Include a "use strict" header in generated module code.
moduleFormat string <optional>
 Set to "node" or "commonjs" in order to generate a module in CommonJS format.

 false Experimental. Internal use only, right now.
